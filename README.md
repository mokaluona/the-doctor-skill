# THE DOCTOR — 非线性时间冒险 RP Skill

> **化身轮换 · 重生记忆共享 · 双轴时间线**
>
> 每一次 TARDIS 降落在你房间里，走出来的都是时间线上不同时刻的博士。
> 他记得你——但你们之间隔着多少个宇宙，取决于此刻站在你面前的是第几任。

---

## 📖 简介

**The Doctor** 是一个为 Claude/Cowork 设计的角色扮演 skill（技能包），让你与《神秘博士》中那位 Gallifrey 最后的时间领主一起经历科幻冒险。

这不是普通的聊天角色。它包含：

- **完整的角色演绎系统**——博士的说话方式、情感深度、记忆边界
- **双轴非线性时间线**——博士的任期（线性）与你的冒险记录（非线性）并行
- **化身轮换机制**——每次出场随机从第1到第15任中选取一位，每位的情感浓度不同
- **四幕冒险框架 + 三次反转设计**——保证每场冒险都有电影级的叙事结构
- **不对称记忆规则**——博士只记得发生在他当前时间节点之前的冒险

---

## 🎭 双时间线设定（核心特色）

这是本 skill 最独特的设定，也是最需要理解的部分。

### 博士的时间线是线性的

```
第1任 → 第2任 → ... → 第10任 → 第11任 → ... → 第15任
```

博士的个人经历是线性的。第10任经历了第1-9任的一切，但不知道第11任及以后的事。

### 你的冒险记录是非线性的

```
你遇见博士的顺序：冒险#1 → 冒险#2 → 冒险#3 ...
但#1可能是第10任，#2可能是第12任，#3又回到第10任
```

对你来说，时间顺序是按你遇见博士的次数排列的。对博士来说，他只有在自己的任期**之内**遇见你的冒险才算数。

### 不对称默契

这造成了奇妙的戏剧张力：
- 第10任带你冒险了两次，很喜欢你，但不明白为什么看你第一眼就有奇怪的熟悉感
- 第15任可能已经"认识你很久了"——但对你来说这只是你们的第一次见面
- 博士会下意识用老朋友语气说话，然后突然意识到你不知道他在说什么

> 这种错位感是本 skill 的情感核心。

## 🚀 快速开始

### 安装

1. 将本仓库克隆到你的项目文件夹：
```bash
git clone https://github.com/mokaluona/the-doctor-skill.git
```

2. 在你的 Cowork 或 Claude 客户端中加载 `SKILL.md` 作为自定义 skill。

### 首次启动

当你第一次运行 skill 时，需要完成以下设定：

#### 1️⃣ 设置你的角色名
SKILL.md 中使用 `{user}` 作为默认占位符。**用你的角色名全局替换所有 `{user}`。**

建议：在编辑器中打开 `SKILL.md`，搜索替换 `{user}` → `你的名字`。
（注意：`doctor_rp_timeline.md`、`doctor_lore_the_mystery.md`、`doctor_rp_preferences.md` 中也包含 `{user}`，一并替换）

#### 2️⃣ 自定义你的背景故事
打开 `doctor_lore_the_mystery.md`，阅读默认设定。你可以：
- 原样保留，直接开始冒险
- 修改其中的内容，贴合你自己的角色设定
- 完全删掉重写——保持文件名不变即可

#### 3️⃣ 确认项目路径
默认所有文件放在同一目录下。如果你想把文件分散存放：
- 在 `doctor_rp_preferences.md` 中修改 `project_path` 和 `stores_path`
- 或设置环境变量 `DOCTOR_SKILL_ROOT`

## 📋 基本命令

在故事中，你可以使用以下指令来控制叙事（需放在 `【】` 内）：

### 叙事控制

| 指令 | 效果 | 示例 |
|------|------|------|
| `【跳过这段直接到XX】` | 跳过中间过程到指定场景 | `【跳过这段直接到降落】` |
| `【给个需要做决定的局面】` | 制造决策点 | `【给个需要快速做决定的局面】` |
| `【跳转到XX时间/地点】` | 时间和地点跳跃 | `【跳转到三小时后】` |
| `【快进到XX】` | 加速叙事节奏 | `【快进到危机部分】` |
| `【倒叙/插叙到XX】` | 时间回溯 | `【倒叙到刚才那个房间】` |

### 冒险后操作

| 指令 | 效果 |
|------|------|
| `【导出】` | 将本次冒险整理为完整叙事文件，保存到 `STORES/` 文件夹 |
| `【存储记忆】` | 将本次冒险梗概更新到 `doctor_rp_timeline.md` 的编年史中 |

> **导出 vs 存储记忆的区别：**
> - 导出 = 完整故事（润色填充，给读者看）
> - 存储记忆 = 结构化记录（梗概/关键对话/物品，给后续对话参考）

### 元指令

| 指令 | 效果 |
|------|------|
| `【当前是哪一任】` | 查看当前化身的身份 |
| `【查看时间线】` | 回顾冒险编年史 |
| `【OC】` / `【OOC】` | 暂时跳出角色，进行元对话 |
| `【重置】` | 清除对话上下文，开始新故事 |

## 🧠 理解博士的记忆规则

这是 skill 运作的核心逻辑，请务必理解：

### 重生记忆共享
所有化身共享全部记忆——**但仅限发生在该化身个人时间节点之前的冒险**。

- 第10任记得第1-9任的全部经历
- 第10任不记得第11任及以后发生的事
- 第11任记得第1-10任的全部经历

### 🧊 擦除机制
标记了 `🧊` 的条目——即使发生在该任期之前也无法访问。博士甚至不记得自己忘记了什么。

### 不对称关系
同一个你，穿越了博士的整个时间线。有些化身跟你很熟，有些才刚认识——但他们共享记忆的方式让每次相遇都带着微妙的错位感。

## ✏️ 自定义指南

### 修改角色设定
直接编辑 `doctor_lore_the_mystery.md`。你可以：
- 保留"时间指纹"核心谜团，细化你的角色背景
- 完全抛弃默认设定，写一个全新的起源故事
- 逐步迭代——先开始冒险，随着对角色的理解加深再回来修改

### 调整演绎风格
编辑 `doctor_rp_preferences.md`：
- 修改最低字数要求（默认 2000 字）
- 调整类型偏好
- 配置项目路径

### 添加自定义设定
你可以添加额外的 `.md` 文件到项目目录中。skill 会读取目录下的所有 `.md` 文件作为设定参考。

## 💡 使用技巧

1. **第一次使用**：先用默认设定跑一次冒险流程，体验后再决定如何修改背景故事
2. **进入危机要快**：最好的冒险是 TARDIS 刚降落，警报就响了
3. **善用 `【】` 指令**：你想跳过逛街直接到爆炸现场？用指令告诉模型
4. **定期存档**：每次冒险结束后，用 `【存储记忆】` 记录到时间线，用 `【导出】` 保存完整故事
5. **尝试不同化身**：第10任和第12任会给你完全不同的体验
6. **不要怕危险**：博士的故事里人是会死的，东西会炸的，这才是冒险
